# README

## LogLite
**LogLite-b:** A lightweight compression method proposed by us, outputting bitstreams and offering the best compression ratio and speed for line-by-line compression.

**LogLite-B:** A lightweight compression method proposed by us, outputting bytestreams, which can be further compressed by general-purpose compression algorithms.

**LogLite-BZ** and **LogLite-BL:** Extensions of LogLite-B, where Zstd and LZMA are applied for further compression, respectively.


## Compilation

To compile the program, use the following commands:

```bash

sudo apt update

sudo apt install libboost-all-dev

```

```bash

cd LogLite-b/   OR    cd LogLite-B/


g++-9 -Ofast -march=native -fdiagnostics-color=always -g ./src/compress/*.cc ./src/common/*.cc ./src/tools/*.cc -I ./src -o ./src/tools/xorc-cli -mavx512f

```

Make sure to link against the necessary Boost libraries ( `boost_system` and `boost_filesystem` ).

## Usage

The utility can be run from the command line with various options:

### Options

- `--compress`: Enable compression mode.
- `--decompress`: Enable decompression mode.
- `--test`: Run both compression and decompression in test mode.
- `--file-path <path>`: Specify the path to the input file.
- `--com-output-path <path>`: Specify the path for the compressed output file.
- `--decom-output-path <path>`: Specify the path for the decompressed output file.


### Example Commands

#### Compression

```bash

./src/tools/xorc-cli --compress --file-path <input_file_path> --com-output-path <compressed_output_file_path>

```

```bash

./src/tools/xorc-cli --compress --file-path data/path/Apache.log --com-output-path data/path/Apache.log.lite

```


#### Decompression

```bash

./src/tools/xorc-cli --decompress --file-path <input_compressed_file_path> --decom-output-path <decompressed_output_file_path>

```

```bash

./src/tools/xorc-cli --decompress --file-path data/path/Apache.log.lite --decom-output-path  data/path/Apache.log.decom

```

#### Test Mode

Run both compression and decompression in test mode.

```bash

./src/tools/xorc-cli --test --file-path <input_file_path> --com-output-path <compressed_output_file_path> --decom-output-path <decompressed_output_file_path>

```

```bash

./src/tools/xorc-cli --test --file-path data/path/Apache.log --com-output-path data/path/Apache.log.lite --decom-output-path data/path/Apache.log.lite.decom

```


#### Further compression

Files generated by LogLite-B can be further compressed using Zstd or LZMA.

To ensure objectivity, we used the most widely used tool, LZBench [https://github.com/inikep/lzbench](https://github.com/inikep/lzbench), for compression.

an end to end implement will be given in the open-source future version.


```bash
lzbench -ezstd,3/lzma,6 <compressed_output_file_path>

lzbench -ezstd,3/lzma,6 data/path/Apache.log.lite
```

## Input Data Format
LogLite is line-wise compression.
Thus the data should be that each line store a string.
Note the line separator: on Linux, it is \n, while on Windows, it is \r\n.
Example:

```bash
string1
string2
string3
...
```

### Baselines

#### Line-by-line Compression

PBC: [https://github.com/antgroup/pbc](https://github.com/antgroup/pbc)
FSST: [https://github.com/cwida/fsst](https://github.com/cwida/fsst)
LZ4-d and Zstd-d:  
- Similar to FSST, we implemented LZ-dict and Zstd-dic in the linetest-zstd.cpp file under the LogLite-b/src/additional directory.
- Download FSST，Copy the linetest-zstd.cpp file into the paper directory of FSST.

```bash
cd fsst/paper/

g++ -std=c++14 -W -Wall -fpermissive -olinetest-zstd -Izstd -Lzstd -g -O3 -DNDEBUG linetest-zstd.cpp -llz4 -lzstd -I.. -L.. -lfsst

./linetest-zstd lz4dict <input_file_path>
./linetest-zstd zstddict <input_file_path>

./linetest-zstd lz4dict data/path/Apache.log.lite
./linetest-zstd zstddict data/path/Apache.log.lite

```
